# 03 — Godot 4 C# Source Generator Integration

## How Godot Uses Source Generators Internally

Godot 4 **replaced all reflection-based interop** with compile-time source generators. When you write a Godot C# node script, `Godot.NET.Sdk` implicitly references the engine's internal generator package. The generator detects `[Export]`, `[Signal]`, `[GlobalClass]`, and similar attributes and emits:

- `_BindMethods()` — registers exported properties and signals with the engine
- `InvokeGodotClassMethod()` — dispatches virtual method calls from the engine
- `GetGodotPropertyList()` / `GetGodotPropertyValue()` etc. — property marshalling glue

Because of this, **all Godot node scripts must be declared `partial`**:
```csharp
// Required — Godot generator emits the other half of this class
public partial class Player : CharacterBody2D { }
```

### Viewing Godot's Generated Code (Development)
```xml
<!-- In your game .csproj -->
<PropertyGroup>
  <EmitCompilerGeneratedFiles>true</EmitCompilerGeneratedFiles>
  <CompilerGeneratedFilesOutputPath>.generated</CompilerGeneratedFilesOutputPath>
</PropertyGroup>
```

## Why Custom Generators Are Essential for Godot

| Problem | Reflection-based fix | Generator-based fix |
|---------|----------------------|---------------------|
| Node binding | `GetNode<Button>("%MyButton")` | `[SceneTree]` partial class with typed properties |
| Input strings | `"ui_accept"` magic strings | Strongly-typed `InputActions.UiAccept` constant |
| Autoload access | `GetNode<MyAutoload>("/root/MyAutoload")` | Static `Autoloads.MyAutoload` property |
| State machines | `Dictionary<State, Action>` at runtime | Compile-time enum dispatch with partial methods |
| Reflection | Crashes on iOS/console AOT export | Generator replaces reflection at compile time |

---

## The AOT / Trimming Constraint

Godot 4 mobile and console exports use **.NET NativeAOT or Mono AOT**. Runtime reflection is either forbidden or produces unpredictable trim warnings:

```
IL2026: Using member 'System.Type.GetProperties()' which has
        'RequiresUnreferencedCodeAttribute' ...
```

**Any architecture that uses `Type.GetProperties()`, `Activator.CreateInstance()`, or dynamic dispatch must be replaced with a source generator before shipping on iOS, Android, or consoles.**

## Recipe 1: `.tscn` Auto-Binder (Scene Node Wiring)

### The Problem
```csharp
// Fragile — string typos crash at runtime; no rename refactoring
public override void _Ready()
{
    var health = GetNode<ProgressBar>("%PlayerHealth");
    var label  = GetNode<Label>("UI/ScoreLabel");
}
```

### Generator Architecture
1. Mark `.tscn` files as `AdditionalFiles` in `.csproj`:
   ```xml
   <ItemGroup>
     <AdditionalFiles Include="**/*.tscn" />
   </ItemGroup>
   <ItemGroup>
     <CompilerVisibleItemMetadata Include="AdditionalFiles" MetadataName="SourceItemType" />
   </ItemGroup>
   ```

2. The generator reads scene files in the pipeline:
   ```csharp
   IncrementalValuesProvider<SceneModel> scenes = context.AdditionalTextsProvider
       .Where(static f => f.Path.EndsWith(".tscn"))
       .Select(static (file, ct) => ParseTscn(file, ct))
       .Where(static m => m is not null)!;
   ```

3. Emit a typed partial class:
   ```csharp
   // Generated: Player.g.cs
   public partial class Player : CharacterBody2D
   {
       private global::Godot.ProgressBar? _playerHealth;
       public global::Godot.ProgressBar PlayerHealth
           => _playerHealth ??= GetNode<global::Godot.ProgressBar>("%PlayerHealth");

       private global::Godot.Label? _scoreLabel;
       public global::Godot.Label ScoreLabel
           => _scoreLabel ??= GetNode<global::Godot.Label>("UI/ScoreLabel");
   }
   ```

### Existing NuGet: `GodotSharp.SourceGenerators`
- `[SceneTree]` attribute does exactly this and more, including traversal of instanced sub-scenes.
- `[GodotOverride]` allows virtual overrides without method collision conflicts.
- `[Notify]` generates property-changed boilerplate compatible with Godot signals.
- `[InputMap]` / `[LayerNames]` read `project.godot` for strongly-typed access.

## Recipe 2: Input Map Constants

### Generator Architecture
1. Read `project.godot` as `AdditionalFile`
2. Parse the `[input]` section for `action_name` keys
3. Emit:
   ```csharp
   // Generated: InputActions.g.cs
   // <auto-generated />
   #nullable enable
   internal static partial class InputActions
   {
       public static readonly global::Godot.StringName UiAccept   = "ui_accept";
       public static readonly global::Godot.StringName UiCancel   = "ui_cancel";
       public static readonly global::Godot.StringName MoveLeft   = "move_left";
       public static readonly global::Godot.StringName MoveRight  = "move_right";
   }
   ```

### Usage
```csharp
// Compile-time safe — rename refactoring works
if (Input.IsActionPressed(InputActions.MoveLeft)) { ... }
```

## Recipe 3: Autoload / Singleton Accessor

### Generator Architecture
1. Parse `project.godot` `[autoload]` section
2. Emit:
   ```csharp
   // Generated: Autoloads.g.cs
   internal static partial class Autoloads
   {
       private static global::MyGame.GameManager? _gameManager;
       public static global::MyGame.GameManager GameManager
           => _gameManager ??= ((global::Godot.Node)
               global::Godot.Engine.GetMainLoop())
               .GetNode<global::MyGame.GameManager>("/root/GameManager");
   }
   ```

## Recipe 4: Compile-Time State Machines (Advanced / .NET 10)

### Architecture Using `partial` Methods + C# 14 `partial` Properties

```csharp
// User writes:
[GenerateStateMachine]
public partial class EnemyAI : Node
{
    [State(initial: true)]
    private partial void Idle();

    [State(transitions: new[] { nameof(Attack), nameof(Flee) })]
    private partial void Chase();

    [State]
    private partial void Attack();

    [State]
    private partial void Flee();
}
```

The generator emits:
- An internal `EnemyState` enum with one value per `[State]` method
- A `CurrentState` partial property backed by the enum
- A `TransitionTo(EnemyState next)` method that validates transitions at runtime (or fully at compile time using interceptors)
- The partial method bodies enforcing entry/exit hooks

### .NET 10 Interceptors (Experimental)
Using C# 14 interceptors, a generator can **silently redirect** a call like:
```csharp
myEnemy.TransitionTo("Attack"); // intercepted, replaced with typed call
```
to:
```csharp
myEnemy.TransitionTo(EnemyState.Attack); // emitted by generator
```
See `04_dotnet10_features.md` for interceptor implementation details.

## Recipe 5: AOT-Safe Resource Loader

### The Problem
```csharp
// Uses reflection, breaks AOT
var scene = GD.Load<PackedScene>("res://Enemies/Orc.tscn");
```

### Generator Approach
Scan all `res://` asset paths referenced in C# source files using `AdditionalTextsProvider`, then emit:
```csharp
internal static partial class Assets
{
    private static global::Godot.PackedScene? _orc;
    public static global::Godot.PackedScene Orc
        => _orc ??= global::Godot.ResourceLoader.Load<global::Godot.PackedScene>(
               "res://Enemies/Orc.tscn");
}
```

## Godot-Specific Pitfalls for Generator Authors

### Pitfall 1: `_Ready()` Method Collision
Godot's own generator may emit a `_Ready()` override. If your custom generator also emits `_Ready()` in the same partial class, the consumer gets a compile error.

**Fix**: Emit a `partial void OnGeneratorReady()` method and have the generator call it from a `_Ready()` only if Godot hasn't claimed it, OR use `partial` method chains:
```csharp
// Generated partial method declaration
partial void OnGeneratorReady();

// Generated _Ready (only if the user's class doesn't already have one)
public override void _Ready()
{
    base._Ready();
    OnGeneratorReady();  // user implements this
}
```

### Pitfall 2: Godot Type Resolution in Tests
When running generator unit tests outside of a Godot project, `Godot.Node` etc. won't be in the compilation. Use:
```csharp
var compilation = CSharpCompilation.Create("TestAssembly",
    references: Basic.Reference.Assemblies.Net80.All  // or include GodotSharp.dll
    // OR add a fake stub:
);
```
Or stub Godot types inline in the test source string:
```csharp
const string GodotStubs = """
    namespace Godot {
        public class Node { }
        public class Node2D : Node { }
        public sealed class ExportAttribute : System.Attribute { }
    }
    """;
```

### Pitfall 3: Generator Ordering
Godot's generator must run before your custom generator if your generator inspects Godot-generated partial members. Set build ordering constraints if needed via MSBuild, or design your generator to operate on the user-authored partial members only.

### Pitfall 4: No GDExtension Bindings for C#
As of Godot 4.x, C# **cannot call GDExtension APIs directly**. Custom generators cannot bridge this gap — you must either use GDScript as an intermediary or wait for the planned GDExtension-based C# rewrite. Design generators to work with the GodotSharp scripting layer only.
